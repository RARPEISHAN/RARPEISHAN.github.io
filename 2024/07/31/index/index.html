Creational Patterns

Creational design patterns are composed of two dominant ideas. One is encapsulating knowledge about which concrete classes the system uses. Another is hiding how instances of these concrete classes are created and combined. –WIKI

Creational Patterns 又分类成 object-creational patterns（主要关注对象的创建） 和 class-instantiation patterns（主要关注类的实例化）

Factory Patterns

Factory Patterns ，个人认为按照工厂这个概念，将功能划分为产品生产与产品使用。

在父类中提供一个创建对象的方法 (依赖于多态的机制)，允许子类决定实例化对象的类型（将构造推迟到到子类中，子类对父类 createProduct() 方法进行重写，从而实现不同产品类型的创建）。

并不一定是创建新的实例。工厂方法也可以返回缓存、对象池或其他来源的已有对象

工程模式实现方式：

  • 让所有产品都遵循同一接口（Product_interface）。该接口必须声明对所有产品都有意义的方法
  • 在创建者(Creator)中创建一个工厂方法(createProduct)。工厂方法返回类型必须遵循通用的产品接口（Product）
  • 每次增加一个产品，重写工厂方法来构建相应的产品（产品过多可以在子类复用基类的控制参数，再分一层抽象）。

Singleton Method Design Pattern

确保一个类只能有一个实例对象，并提供一个全局的访问点。

  1. 单列模式初始化方式：Early initialization（即使不请求也初始化） 和 lazy initialization（请求既初始化）。
  2. 单例设计模式中，构造函数权限都是私有的，也就导致了无法被任何类继承（因为没办法构造啊）。
