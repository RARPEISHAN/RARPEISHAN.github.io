CMake

CMake 是个一个开源的跨平台自动化建构系统，用 配置文件控制建构过程（build process）的方式管理程序，并不依赖于某特定编译器，并可支持多层目录、多个应用程序与多个函式库。 它–Wiki

CMake 语法介绍

PROJECT

用来指定工程的名字与支持的语言的行为

写法 行为
PROJECT(HELLO) 指定文件名，并支持所有语言
PROJECT(HELLO CXX) 指定文件名，支持 c++语言
PROJECT(HELLO C CXX) 指定文件名，并支持 c/c++语言

该指定隐式定义了两个变量（之间等价）

MESSAGE

向用户输出自定义的信息

写法 行为
SEND_ERROR 产生错误，built 过程被跳过
STATUS 输出前提为 -- 的信息
FATAL_ERROR 立即终止所有 cmake 过程

若 FATAL_ERROR 在 SEND_ERROR 前，之后的操作都不会执行。

SET

用来指定变量

参数 作用
EXECUTABLE_OUTPUT_PATH 最终可执行文件存放目录
LIBRARY_OUTPUT_PATH 最终的库文件（静态库、共享库）存放的目录

ADD_EXECUTABLE

生成可执行文件

写法 行为
ADD_EXECUTABLE(hello ${SRC_LIST}) 对变量{SRC_LIST}中的源文件进行读写，并生成可执行文件 hello

ADD_LIBRARY

CMake 中生成静态库或动态库

参数 作用
lib_name 指定生成库的名称
SHARED 生成动态库
STATIC 生成静态库
MODULE 使用 dyld ( the dynamic link editor · 苹果的动态连接器) 的系统有效，如果不支持 dyld，则为 SHARED
EXCLUDE_FROM_ALL 指定此参数，则该库默认情况下不被构建，当有其他组件用到该库或手动构建时，才被构建
src1 src2 …… srcN 用于构建 库用到的源文件

INCLUDE_DIRECTORIES

用于增加包含头文件的路径

TARGET_LINK_LIBRARIES

指定某个目标生成所依赖的库

ADD_SUBDIRECTORY

CMake 提供 ADD_SUBDIRECTORY 指令，用于向当前工程添加子目录，并且可以 指定中间二进制和目标二进制文件 存放的位置，还可以将指定目录排除出编译过程

参数 作用
src_dir 把 src_dir 这个子目录加入工程
bin_dir 指定编译输出（包含编译的中间结果）的目录为 bin_dir 同名的目录；指定就是把 src_dir 重命名为 bin_dir
EXCLUDE_FROM_ALL 将给定的目录从编译中排除，既不编译它，比如一些示例的目录，需要等工程构建完后单独构建

add_subdirectory 指令可以指定编译输出目录，也可以通过 set 指令来设置 executable_output_path 和 library_output_path 指定可执行文件的输出

生成执行文件的基本过程

需要明白一个生成执行文件的构建过程

预处理

  1. 处理 #include 预编译指令

  将被包含的 文件的内容 插入到该预编译指令的位置 （递归）

  1. 宏定义替换

  2. 处理所有预编译指令，例如 #ifndef、#ifdef、#endif、#elif、#else

  3. 删除所有注释 5.添加行号与文件标识

  为了方便编译时期产生编译错误时显示行号

  1. 保留所有 #pragma

处理.c/.h 文件预处理后所得文件 .i ，而 .cpp/cxx/hpp 预处理后所得文件 .ii.

编译阶段

编译阶段就是把预处理完的文件进行一系列词法分析、语法分析、语义分析及优化生产相应的汇编代码文件，这个过程往往是我们所说的整个程序构建的核心部分。

  • 词法分析： 程序员通过扫描器（如 lex 程序，一个独立的的词法扫描器，方便）
  • 语法分析
  • 语义分析
  • 中间代码生成器
  • 代码优化器
  • 目标代码生成器

依旧参考 a.c 这个文件，经过编译阶段后生成的汇编指令

汇编阶段

将汇编代码转变成机器可以执行的指令，每一个汇编语句几乎都对应一条机器指令。

链接阶段

程序设计的模块化是人们一直在追求的目标，以此把每个源代码模块独立地编译，然后按照需要将它们“组装”起来，这就是链接。链接程序将目标文件、库和运行库合并生成可执行文件。

函数库包含 运行时库 与 静态时库。预处理中只是对引入了头文件申明，而没有函数的实现，链接则是实现函数调用的处理（地址重定位）。

  • 地址空间和分配 （Address and Storage allocation）

  扫描所有输入的目标文件，获得各个段长度、属性和位置，并输入目标文件符号表中所有符号定义与引用，统一放到全局符号表。这一步将获得所有输入目标文件的段长度，并合并（以上都为空间分配），计算出输入文件中各个段合并后的长度与位置，并建立映射长度（以上为地址分配）。

  • 符号决议 （Symbol Resolution/Binding）与 符号的重定向

  读取输入文件中段的数据、重定位信息，并且进行符号解析与重定位、调整代码中的地址等。(这里实现了外部符号的引入)
